# CRUD Backend для заметок

Простой REST API сервер для управления заметками, реализованный на **Express.js**.

## Описание

Сервер предоставляет CRUD-операции (Create, Read, Update, Delete) для работы с заметками. Данные хранятся в оперативной памяти (массив в памяти процесса).

## Технологии

- **Node.js** >= 16.0.0
- **Express.js** 4.18.2 — веб-фреймворк
- **CORS** 2.8.5 — поддержка кросс-доменных запросов
- **body-parser** 1.20.2 — парсинг JSON-тел запросов

## Установка

```bash
npm install
```

## Запуск

```bash
npm start
```

Сервер запустится на порту **7070** (или на порту из переменной окружения `PORT`).

## API

| Метод | Endpoint | Описание |
|-------|----------|----------|
| `GET` | `/notes` | Получить все заметки |
| `POST` | `/notes` | Создать новую заметку |
| `DELETE` | `/notes/:id` | Удалить заметку по ID |

### Примеры запросов

#### Получить все заметки

```http
GET http://localhost:7070/notes
```

#### Создать заметку

```http
POST http://localhost:7070/notes
Content-Type: application/json

{
  "content": "Текст заметки"
}
```

#### Удалить заметку

```http
DELETE http://localhost:7070/notes/3
```

## Структура данных

Заметка имеет следующую структуру:

```json
{
  "id": 1,
  "content": "Текст заметки"
}
```

- `id` — уникальный идентификатор (присваивается автоматически)
- `content` — содержимое заметки

## Тестирование

Для тестирования API можно использовать файл `api.rest` в сочетании с расширением REST Client для VS Code или аналогичными инструментами (Postman, Insomnia).

## Особенности

- Данные не сохраняются после перезапуска сервера (in-memory хранилище)
- Сервер возвращает статус `204 No Content` для операций `POST` и `DELETE`
- Поддерживаются кросс-доменные запросы благодаря CORS
